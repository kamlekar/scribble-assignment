# Implementation Plan: Drawing Canvas

**Branch**: `003-drawing-canvas` | **Date**: 2026-06-11 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/003-drawing-canvas/spec.md`

## Summary

Implement an interactive HTML5 drawing canvas for the assigned drawer with colour palette and brush width selection. Strokes are synced to guessers via HTTP polling every 2 seconds. The canvas is per-round, cleared automatically at round start. Stroke data is stored in-memory as vector metadata (points, colour, width) keyed by client-generated UUID.

## Technical Context

**Language/Version**: TypeScript (ES2022+, strict mode)

**Primary Dependencies**: React 18 (canvas rendering), Express (backend API), Zod (payload validation), Vitest (testing)

**Storage**: In-memory — `CanvasState` held in the Room model on the backend. No database.

**Testing**: Vitest for both backend and frontend. Component tests for canvas rendering, integration tests for stroke sync flow.

**Target Platform**: Web (modern Chrome, Firefox, Safari). Mouse input only (no touch).

**Project Type**: Web application — Express backend + React frontend

**Performance Goals**: Stroke appears on guessers' canvases within 2 seconds of mouse-up. Full drawing state fetch for mid-round joiners under 2 seconds.

**Constraints**: No WebSockets (HTTP polling only, 2s interval). No database (in-memory only). No authentication. Rooms cleaned up when empty.

**Scale/Scope**: Party game — <20 concurrent users per room, single room per process.
