# Data Model: Lobby & Host Management

## Entities

### Participant

| Field | Type | Description | Validation |
|-------|------|-------------|------------|
| `id` | `string` (UUID) | Unique participant identifier | Generated by `crypto.randomUUID()` |
| `name` | `string` | Display name, trimmed | `min(1)`, `max(30)` after trim, no empty/whitespace |
| `joinedAt` | `string` (ISO 8601) | Join timestamp | `new Date().toISOString()` |

**Changes from current model**: No structural change to `Participant` — `isHost` is derived at snapshot time from `room.hostId`.

### Room

| Field | Type | Description | Validation |
|-------|------|-------------|------------|
| `code` | `string` (4-char) | Unique room code | Generated by `generateUniqueCode()` (no ambiguous chars) |
| `hostId` | `string` (UUID) | Participant ID of the host | Must match a participant in the room |
| `status` | `RoomStatus` | Current room phase | `"lobby"` | `"playing"` | `"finished"` |
| `participants` | `Participant[]` | All participants in the room | At least 1 |
| `word` | `string` | Current round's word (set on game start) | Selected from `STARTER_WORDS` |
| `createdAt` | `string` (ISO 8601) | Room creation timestamp | `new Date().toISOString()` |
| `updatedAt` | `string` (ISO 8601) | Last mutation timestamp | Updated on every save |

**Changes from current model**:
- Added `hostId: string` (new)
- Changed `status` from `RoomStatus = "lobby"` to `RoomStatus = "lobby" | "playing" | "finished"` (extended)
- Added `word: string` (new)

### LobbySnapshot

| Field | Type | Description |
|-------|------|-------------|
| `code` | `string` | Room code |
| `status` | `RoomStatus` | Current status |
| `participants` | `ParticipantSnapshot[]` | Participant list with host indicator |
| `hostId` | `string` | Which participant is host |

### ParticipantSnapshot

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Participant ID |
| `name` | `string` | Display name |
| `joinedAt` | `string` | Join timestamp |
| `isHost` | `boolean` | Whether this participant is the host (derived: `id === room.hostId`) |
| `role` | `ParticipantRole` | `"drawer"`, `"guesser"`, or `null` in lobby |

## State Transitions

```
Room Status Flow:
  lobby ──(host starts game)──> playing ──(round ends)──> finished
```

```
Host Transitions:
  Player creates room ──> hostId = creator.id
  Host leaves room ──> hostId = remaining[0].id (earliest joined)
  Last participant leaves ──> room deleted
```

## Validation Rules

| Rule | Location | Enforcement |
|------|----------|-------------|
| Player name must be non-empty after trim | Backend (Zod) + Frontend | `z.string().trim().min(1)` |
| Player name max 30 chars | Backend (Zod) + Frontend | `z.string().max(30)` |
| Room code must exist for join | Backend | `roomStore.getRoom(code)` returns non-null |
| Room must not be in "playing" state for join | Backend | Check `room.status === "lobby"` before join |
| Only host can start game | Backend | Verify `participantId === room.hostId` |
| Need at least 2 players to start | Backend | `room.participants.length >= 2` |
| Concurrency: requests atomic via Zod | Backend | Validate input before mutating room state — no partial writes |

## Existing Types Not Modified

- `ParticipantRole`: remains `"drawer" | "guesser"` (no change)
- `RoomSessionResponse`: remains `{ participantId, room: RoomSnapshot }` (no change)
