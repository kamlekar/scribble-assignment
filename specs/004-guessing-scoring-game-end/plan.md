# Implementation Plan: Guessing, Scoring & Game End

**Branch**: `004-guessing-scoring-game-end` | **Date**: 2026-06-12 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/004-guessing-scoring-game-end/spec.md`

## Summary

Implement guess submission with validation, correct-guess detection, scoring, round result display, and restart-to-lobby flow. All sync uses HTTP polling (single endpoint returning full room state). Only the first correct guesser receives points (100 pts). The drawer does not receive points in v1.

## Technical Context

**Language/Version**: TypeScript (ES2022+, strict mode)

**Primary Dependencies**: React 18, Express, Zod (validation), Vitest (testing)

**Storage**: In-memory — guesses, scores, and round results held in the Room model on the backend. No database.

**Testing**: Vitest for both backend and frontend. Manual validation per quickstart.md.

**Target Platform**: Web (modern Chrome, Firefox, Safari). Mouse and keyboard input.

**Project Type**: Web application — Express backend + React frontend

**Performance Goals**: Guess appears in shared history within 2 seconds. Result screen visible within 2 seconds of correct guess. Restart completes within 3 seconds.

**Constraints**: No WebSockets (HTTP polling only, 2s interval). No database (in-memory only). No authentication. Rooms cleaned up when empty.

**Scale/Scope**: Party game — <20 concurrent users per room, single room per process.
