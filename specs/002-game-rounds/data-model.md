# Data Model: Game Rounds

## Entities

### Participant (no change to base model)

| Field | Type | Description | Validation |
|-------|------|-------------|------------|
| `id` | `string` (UUID) | Unique participant identifier | Generated by `crypto.randomUUID()` |
| `name` | `string` | Display name, trimmed | `min(1)`, `max(30)` after trim |
| `joinedAt` | `string` (ISO 8601) | Join timestamp | `new Date().toISOString()` |

**Note**: `Participant.role` remains a transient field; it is computed at snapshots, not stored long-term. See Room.roles below.

### ParticipantSnapshot (added role population)

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Participant ID |
| `name` | `string` | Display name |
| `joinedAt` | `string` | Join timestamp |
| `isHost` | `boolean` | Derived: `id === room.hostId` |
| `role` | `ParticipantRole` | `"drawer"` for host when playing, `"guesser"` for others, `null` in lobby |

**Changes from 001**:
- `role` is now populated: `"drawer"` if participant is the host and status is `"playing"`/`"finished"`, `"guesser"` for all others, `null` for lobby

### Room (no structural changes to fields)

| Field | Type | Description |
|-------|------|-------------|
| `code` | `string` (4-char) | Unique room code |
| `hostId` | `string` (UUID) | Participant ID of the host |
| `status` | `RoomStatus` | `"lobby"` | `"playing"` | `"finished"` |
| `participants` | `Participant[]` | All participants |
| `word` | `string` | Current round's secret word (set on game start) |
| `createdAt` | `string` (ISO 8601) | Room creation timestamp |
| `updatedAt` | `string` (ISO 8601) | Last mutation timestamp |

**No structural changes from 001**. The `word` and `status` fields already exist.

### RoomSnapshot (role-aware filtering)

| Field | Type | Visibility | Notes |
|-------|------|------------|-------|
| `code` | `string` | All | |
| `status` | `RoomStatus` | All | |
| `participants` | `ParticipantSnapshot[]` | All | |
| `hostId` | `string` | All | |
| `word` | `string` | **Drawer only**. Omitted for guessers. | Not populated in lobby — only set on game start |
| `availableWords` | `string[]` | All | Only present in POST /rooms/:code/start response; omitted from GET /rooms/:code |
| `roles` | `ParticipantRole[]` | All | Only present in POST /rooms/:code/start response; omitted from GET /rooms/:code |

**Changes from 001**:
- `word` is now conditionally included based on `viewerParticipantId`: included if viewer is the drawer, omitted otherwise
- `availableWords` and `roles` are included only in the start-game response (POST), not in the state-polling response (GET)

### ParticipantRole (unchanged from existing)

```
"drawer" | "guesser"
```

## State Transitions

```
Room Status Flow (unchanged from 001):
  lobby ──(host starts game)──> playing ──(round ends)──> finished
```

```
Role Assignment (new):
  Game starts ──> hostId === drawer, all others === guesser
```

## Validation Rules

| Rule | Location | Enforcement |
|------|----------|-------------|
| Drawer assignment must pick exactly one participant | Backend (startGame) | `room.hostId` is always the drawer |
| Word must be from starter list | Backend (startGame) | Selected from `STARTER_WORDS` |
| Word must not leak to guessers | Backend (toRoomSnapshot) | Filtered by viewer role lookup |
| Roles must be populated in snapshot | Backend (toRoomSnapshot) | Set on all ParticipantSnapshot entries when status is `"playing"`/`"finished"` |

## Existing Types Not Modified

- `RoomSessionResponse`: remains `{ participantId, room: RoomSnapshot }` (no change)
- `ParticipantRole` type: remains `"drawer" | "guesser"` (no change)
- `RoomStatus`: remains `"lobby" | "playing" | "finished"` (no change)
